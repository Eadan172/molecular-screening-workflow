# 工作流架构说明

## 整体架构

分子筛选工作流采用模块化设计，包含6个核心阶段，每个阶段独立运行并可单独配置。

```
┌─────────────────────────────────────────────────────────────────┐
│                     分子筛选工作流系统架构                         │
└─────────────────────────────────────────────────────────────────┘

┌──────────────┐      ┌──────────────┐      ┌──────────────┐
│   数据输入    │──────>  工作流引擎   │──────>  结果输出    │
└──────────────┘      └──────────────┘      └──────────────┘
                             │
                             ▼
                    ┌────────────────┐
                    │   配置管理器    │
                    └────────────────┘
                             │
                             ▼
                    ┌────────────────┐
                    │   日志记录器    │
                    └────────────────┘
```

## 模块架构

### 1. RNN分子生成模块

```
┌─────────────────────────────────────────────────────────────┐
│                    RNN分子生成模块                            │
├─────────────────────────────────────────────────────────────┤
│  输入:                                                       │
│    - 训练分子库 (SMILES格式)                                 │
│    - 模型配置参数                                            │
│                                                              │
│  处理流程:                                                   │
│    1. SMILES → SELFIES转换                                   │
│    2. 构建词汇表                                             │
│    3. 序列编码与padding                                      │
│    4. LSTM模型训练                                           │
│    5. 分子生成（温度采样）                                    │
│                                                              │
│  输出:                                                       │
│    - 生成的分子SMILES                                        │
│    - 训练好的RNN模型                                         │
└─────────────────────────────────────────────────────────────┘
```

**技术栈**:
- TensorFlow/Keras
- SELFIES (分子表示)
- LSTM网络架构

**关键参数**:
- `temperature`: 控制生成多样性 (0.1-1.0)
- `top_k`: Top-k采样参数
- `num_molecules`: 生成分子的数量

### 2. QSAR活性预测模块

```
┌─────────────────────────────────────────────────────────────┐
│                    QSAR活性预测模块                           │
├─────────────────────────────────────────────────────────────┤
│  输入:                                                       │
│    - 靶点活性数据 (SMILES + IC50)                            │
│    - 待预测分子                                              │
│                                                              │
│  处理流程:                                                   │
│    1. 分子描述符计算 (RDKit)                                 │
│    2. 特征选择                                               │
│    3. 模型训练 (Random Forest / Ensemble)                   │
│    4. 活性预测                                               │
│    5. 结果筛选                                               │
│                                                              │
│  输出:                                                       │
│    - IC50预测值                                              │
│    - 筛选后的分子列表                                        │
└─────────────────────────────────────────────────────────────┘
```

**技术栈**:
- RDKit (分子描述符)
- scikit-learn (机器学习)
- Ensemble方法

**关键参数**:
- `ic50_threshold`: IC50筛选阈值 (nM)
- `top_n`: 选择前N个分子
- `test_size`: 训练测试分割比例

### 3. ADMET过滤模块

```
┌─────────────────────────────────────────────────────────────┐
│                    ADMET过滤模块                              │
├─────────────────────────────────────────────────────────────┤
│  输入:                                                       │
│    - QSAR筛选后的分子                                        │
│    - ADMET预测结果（外部获取）                                │
│                                                              │
│  处理流程:                                                   │
│    1. 读取ADMET预测结果                                      │
│    2. 应用12项过滤标准                                       │
│    3. 筛选合格分子                                           │
│                                                              │
│  输出:                                                       │
│    - ADMET合格的分子                                         │
└─────────────────────────────────────────────────────────────┘
```

**12项ADMET指标**:
1. P-gp底物 (P-gp substrate)
2. P-gp I型抑制剂 (P-gp I inhibitor)
3. P-gp II型抑制剂 (P-gp II inhibitor)
4. CYP2D6底物 (CYP2D6 substrate)
5. CYP3A4底物 (CYP3A4 substrate)
6. CYP1A2抑制剂 (CYP1A2 inhibitor)
7. CYP2C19抑制剂 (CYP2C19 inhibitor)
8. CYP2C9抑制剂 (CYP2C9 inhibitor)
9. AMES毒性 (AMES toxicity)
10. hERG II型抑制剂 (hERG II inhibitor)
11. 肝毒性 (Hepatotoxicity)
12. 皮肤过敏 (Skin Sensitisation)

### 4. 分子对接模块

```
┌─────────────────────────────────────────────────────────────┐
│                    分子对接模块                               │
├─────────────────────────────────────────────────────────────┤
│  输入:                                                       │
│    - ADMET合格的分子                                         │
│    - 靶蛋白PDB文件                                           │
│    - 对接盒参数                                              │
│                                                              │
│  处理流程:                                                   │
│    1. 配体文件准备 (SDF/PDBQT)                               │
│    2. 受体文件准备                                           │
│    3. 批量对接计算                                           │
│    4. 结果收集与排序                                         │
│                                                              │
│  输出:                                                       │
│    - 对接构象 (PDB格式)                                      │
│    - 结合能评分                                              │
└─────────────────────────────────────────────────────────────┘
```

**支持的对接引擎**:
- AutoDock Vina
- GNINA (深度学习对接)
- DiffDock (扩散模型)

**关键参数**:
- `center_x/y/z`: 对接盒中心坐标
- `size_x/y/z`: 对接盒尺寸
- `exhaustiveness`: 搜索详尽程度

### 5. SA Score评估模块

```
┌─────────────────────────────────────────────────────────────┐
│                    SA Score评估模块                           │
├─────────────────────────────────────────────────────────────┤
│  输入:                                                       │
│    - 对接结果                                                │
│                                                              │
│  处理流程:                                                   │
│    1. 计算合成可及性评分                                     │
│    2. 分子复杂度分析                                         │
│    3. 结果排序                                               │
│                                                              │
│  输出:                                                       │
│    - SA Score评分                                            │
│    - 排序后的候选分子                                        │
└─────────────────────────────────────────────────────────────┘
```

**SA Score说明**:
- 范围: 1-10
- 1-3: 易于合成
- 3-6: 中等难度
- 6-10: 难以合成

## 配置管理

配置文件采用YAML格式，支持：

- 数据路径配置
- 模型参数调整
- 过滤标准设置
- 输出选项控制

## 日志系统

日志系统记录：

- 每个步骤的执行状态
- 耗时统计
- 错误信息
- 中间结果路径

日志文件位置: `logs/workflow_YYYYMMDD_HHMMSS.log`

## 异常处理

工作流包含完善的异常处理机制：

1. **输入验证**: 检查文件是否存在、格式是否正确
2. **依赖检查**: 验证所需模块是否安装
3. **错误捕获**: 捕获并记录异常信息
4. **中间结果保留**: 失败时保留已完成的步骤结果
5. **恢复机制**: 支持从特定步骤继续执行
